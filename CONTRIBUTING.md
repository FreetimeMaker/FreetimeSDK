# Contributing to Freetime Payment SDK

First off, thank you for considering contributing to the Freetime Payment SDK! It's people like you that make this tool better for everyone.

## How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/FreetimeMaker/FreetimeSDK/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce it.

### Suggesting Enhancements
*   Check the [Issues](https://github.com/FreetimeMaker/FreetimeSDK/issues) for existing suggestions.
*   Open a new issue describing your proposed change and why it would be useful.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs, update the documentation.
4.  Ensure the test suite passes.
5.  Make sure your code lints.

## Coding Standards

*   Use Kotlin coding conventions.
*   Keep the SDK F-Droid friendly: **No proprietary binary blobs.**
*   Keep it serverless: Avoid requiring backend infrastructure unless strictly necessary and optional.
